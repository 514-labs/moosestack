## Key Concepts

### WebApp Class

All custom frameworks use the `WebApp` class to mount within your Moose project:

```python
from moose_lib.dmv2 import WebApp, WebAppConfig

webapp = WebApp(
    "myApp",
    app,
    WebAppConfig(mount_path="/myapi")
)
```

### Accessing Data

Access your OLAP database through MooseStack's injected utilities:

```python
from moose_lib.dmv2.web_app_helpers import get_moose_utils

moose = get_moose_utils(request)
client = moose.client
jwt = moose.jwt
```

### Mount Path Rules

Mount paths must follow these rules:
- Cannot be `/` (root path)
- Cannot end with a trailing slash
- Cannot start with reserved paths: `/admin`, `/api`, `/consumption`, `/health`, `/ingest`, `/moose`, `/ready`, `/workflows`
- Must be unique across all WebApps

## Choosing a Framework

**Use Native (Api Class)** when you need:
- Simple GET endpoints with query parameters
- Automatic Pydantic validation
- Automatic OpenAPI documentation
- Minimal setup and configuration

**Use FastAPI** when you need:
- Custom middleware and dependencies
- Advanced routing patterns
- WebSocket support
- Background tasks
- Dependency injection
- Full control over request/response lifecycle
