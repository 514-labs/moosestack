## Key Concepts

### WebApp Class

All custom frameworks use the `WebApp` class to mount within your Moose project:

```ts
import { WebApp } from "@514labs/moose-lib";

export const myApp = new WebApp(name, appOrHandler, {
  mountPath: "/myapi",
  metadata: { description: "My custom API" }
});
```

### Accessing Data

Access your OLAP database through MooseStack's injected utilities:

```ts
import { getMooseUtils } from "@514labs/moose-lib";

const moose = getMooseUtils(req);
const { client, sql, jwt } = moose;
```

### Mount Path Rules

Mount paths must follow these rules:
- Cannot be `/` (root path)
- Cannot end with a trailing slash
- Cannot start with reserved paths: `/admin`, `/api`, `/consumption`, `/health`, `/ingest`, `/moose`, `/ready`, `/workflows`
- Must be unique across all WebApps

## Choosing a Framework

**Use Native (Api Class)** when you need:
- Simple GET endpoints with query parameters
- Automatic type validation and OpenAPI documentation
- Minimal setup and configuration

**Use Express** when you need:
- Large middleware ecosystem
- Compatibility with existing Express applications
- Familiar routing patterns

**Use Fastify** when you need:
- High performance and low latency
- Schema-based validation
- Plugin architecture

**Use Koa** when you need:
- Modern async/await middleware
- Lightweight and modular design
- Fine-grained control

**Use Raw Node.js** when you need:
- Maximum control and minimal dependencies
- Custom request handling logic
- Performance-critical applications
