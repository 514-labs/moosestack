# 514 deployment

Use `514 deployment` to inspect deployments for a project.

Plural alias also works: `514 deployments`.

## Usage

```bash
514 deployment [OPTIONS] <COMMAND>
```

## Subcommands

### List

List deployments for a project.

```bash
514 deployment list [OPTIONS] <PROJECT_REF>
```

Arguments:
- `<PROJECT_REF>`: Project reference as `<org>/<project>`, project name, project ID, or Boreal URL.

Options:
- `--api-url <API_URL>`: API base URL override (`HOSTING_CLI_API_URL`).
- `--branch-id <BRANCH_ID>`: Filter by branch ID.
- `-o, --org <ORG>`: Organization name or ID.
- `--status <STATUSES>`: Filter by deployment status (repeatable).
- `--json`: Output as JSON.
- `--limit <LIMIT>`: Number of results to return (default: `20`).
- `--page-token <PAGE_TOKEN>`: Pagination token from a previous response.
- `-v, --verbose...`: Enable verbose output (`-v` debug, `-vv` trace).
- `-h, --help`: Print help.

## Examples

```bash
514 deployment list my-org/my-project
514 deployment list my-org/my-project --status running --status failed
514 deployments list my-org/my-project --limit 50
```
