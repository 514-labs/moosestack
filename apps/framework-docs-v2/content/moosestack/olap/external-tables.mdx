# External Tables

External tables let you query warehouse resources that Moose does not manage.

## Mark tables as externally managed

### TypeScript

```ts
import { Key, LifeCycle, OlapTable } from "@514labs/moose-lib";

interface Orders {
  id: Key<string>;
  total: number;
}

export const ExternalOrders = new OlapTable<Orders>("orders", {
  lifeCycle: LifeCycle.EXTERNALLY_MANAGED,
});
```

### Python

```python
from pydantic import BaseModel
from moose_lib import Key, LifeCycle, OlapConfig, OlapTable

class Orders(BaseModel):
    id: Key[str]
    total: float

external_orders = OlapTable[Orders](
    "orders",
    OlapConfig(life_cycle=LifeCycle.EXTERNALLY_MANAGED),
)
```

## Sync models

```bash
moose db pull --olap-engine clickhouse --olap-url "$MOOSE_OLAP__CONNECTION"
```

## Best practices

- Keep external models in one generated file.
- Re-run `db pull` when upstream schema changes.
- Keep transforms and APIs in common-core types.
