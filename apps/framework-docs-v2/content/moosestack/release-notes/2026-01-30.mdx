### New Features
- **First-class workflows with infrastructure tracking and lifecycle management** ([@DatGuyJonathan](https://github.com/DatGuyJonathan))
  Workflows are now treated as first-class infrastructure resources in MooseStack. They are tracked in the infrastructure map, displayed in plan diffs, and managed through the standard infrastructure lifecycle. This means workflow changes (additions, removals, configuration updates) are now visible during `moose plan` and properly managed during deployments. Workflows also now persist across shutdowns - they no longer terminate automatically in development mode, providing more consistent behavior between dev and production environments.

- **Add secure password input with masked characters for CLI prompts** ([@callicles](https://github.com/callicles))
  Password inputs in the CLI now show asterisks (*) instead of plain text for better security. This affects commands that prompt for database passwords, such as connecting to remote ClickHouse instances. The password input supports backspace for corrections and Ctrl+C to cancel.

- **Add Next.js dashboard demo with query layer for ClickHouse** ([@okane16](https://github.com/okane16))
  New comprehensive Next.js example showing how to build fast customer-facing dashboards on ClickHouse using MooseStack's query layer. Includes a complete dashboard with metrics, time-series charts, and a dynamic report builder. Demonstrates the query model pattern for rapidly building type-safe API endpoints on OLAP tables, targeting product engineers migrating from Postgres to ClickHouse.

  **Building type-safe ClickHouse queries with defineQueryModel**
  ```typescript
  // Define a query model for your ClickHouse table
  import { defineQueryModel, sql } from "moose";
  import { Events } from "./models";
  
  export const eventsModel = defineQueryModel({
    table: Events,
    dimensions: {
      status: { column: "status" },
      day: { expression: sql`toDate(${Events.columns.event_time})`, as: "time" },
      month: { expression: sql`toStartOfMonth(${Events.columns.event_time})`, as: "time" },
    },
    metrics: {
      totalEvents: { agg: sql`count(*)` },
      totalAmount: { agg: sql`sum(${Events.columns.amount})` },
      avgAmount: { agg: sql`avg(${Events.columns.amount})` },
    },
    filters: {
      timestamp: { column: "event_time", operators: ["gte", "lte"] as const },
      status: { column: "status", operators: ["eq", "in"] as const },
    },
    sortable: ["totalAmount", "totalEvents"] as const,
  });
  
  // Create type-safe query functions
  export async function getEventMetrics(startDate?: Date, endDate?: Date) {
    // Model generates type-safe SQL - invalid fields caught at compile time
    const query = eventsModel.toSql({
      dimensions: [],
      metrics: ["totalEvents", "totalAmount", "avgAmount"],
      filters: {
        timestamp: { gte: startDate, lte: endDate },
      },
    });
    
    return await executeQuery(query);
  }
  
  // Dynamic queries with full type safety
  export async function runEventsQuery(params: typeof eventsModel.$inferRequest) {
    return await eventsModel.query(params, clickhouseClient);
  }
  ```
  
  Shows how to use MooseStack's new query layer to build type-safe API endpoints on ClickHouse tables. The defineQueryModel function creates a semantic layer with dimensions, metrics, and filters that generates SQL while catching invalid queries at compile time.

- **Add SAP HANA CDC connector for real-time data streaming** ([@514Ben](https://github.com/514Ben))
  Added a new SAP HANA Change Data Capture (CDC) connector that enables users to stream real-time data changes from SAP HANA databases to ClickHouse. This connector allows users to capture and sync database changes as they happen, enabling real-time analytics and data processing workflows.

  **SAP HANA CDC streaming pipeline with real-time analytics**
  ```python
  from moose_lib import Moose
  from dataclasses import dataclass
  from typing import Optional
  from datetime import datetime
  
  # Define the data model for SAP HANA changes
  @dataclass
  class CustomerChange:
      customer_id: str
      name: str
      email: str
      status: str
      operation: str  # INSERT, UPDATE, DELETE
      change_timestamp: datetime
      
  app = Moose()
  
  # Configure SAP HANA CDC connector
  @app.streaming_function(
      source="sap-hana-cdc://customers",
      sink="clickhouse://customer_changes"
  )
  def process_customer_changes(change: CustomerChange) -> CustomerChange:
      """
      Stream real-time customer changes from SAP HANA to ClickHouse
      """
      # Add any transformation logic here
      if change.operation == "DELETE":
          change.status = "DELETED"
      
      return change
  
  # Optional: Create aggregations on the streamed data
  @app.aggregation(
      source="customer_changes",
      sink="customer_stats"
  )
  def customer_change_stats():
      return """
      SELECT 
          operation,
          COUNT(*) as change_count,
          toStartOfHour(change_timestamp) as hour
      FROM customer_changes
      GROUP BY operation, hour
      ORDER BY hour DESC
      """
  ```
  
  This example shows how to set up a real-time data pipeline that captures changes from a SAP HANA database using the new CDC connector and streams them to ClickHouse. It includes data model definition, streaming function configuration, and optional aggregations for analytics.
