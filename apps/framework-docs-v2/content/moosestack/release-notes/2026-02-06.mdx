# February 6, 2026

## Moose docs and RFC update

This release publishes a full documentation contract for multi-engine OLAP and remote-first development.

### New

- Accepted RFC: [Multi-Engine and Remote-First Developer Experience](https://github.com/514-labs/moose/blob/main/rfd/0010/README.mdx)
- [Remote-First Quickstart](/moosestack/getting-started/remote-first-quickstart)
- [Start from an Existing Warehouse](/moosestack/getting-started/from-existing-warehouse)
- [Engine and Dev Profile](/moosestack/configuration/engine-and-dev-profile)
- [Engine Capability Matrix](/moosestack/reference/engine-capability-matrix)
- [Multi-Engine Migration Playbooks](/moosestack/migrate/multi-engine-migration-playbooks)
- [Legacy ClickHouse Compatibility](/moosestack/migrate/legacy-clickhouse-compatibility)
- [DuckDB engine guide](/moosestack/engines/duckdb)
- [Snowflake engine guide](/moosestack/engines/snowflake)

### Contract highlights

- Canonical config uses `[olap]` and `[dev_infra]`.
- Canonical CLI uses `--olap-engine`, `--olap-url`, and `--infra-profile`.
- Diagnostics are standardized across engines and profiles.
