**Move workflow routes behind admin authentication** ([@DatGuyJonathan](https://github.com/DatGuyJonathan))
  Workflow API endpoints now require admin authentication for security. The workflow trigger and history endpoints have been moved under `/admin/` paths and require a valid admin bearer token. This affects users who programmatically trigger workflows or query workflow history - they'll need to update their API calls to include proper authentication headers and use the new admin endpoints.

  **Updating workflow API calls for admin authentication**

  ```typescript
  // Before: Workflow operations without authentication
  // const response = await fetch('http://localhost:4000/workflows/my-pipeline/trigger', {
  //   method: 'POST',
  //   body: JSON.stringify({ data: userData })
  // });
  
  // After: Workflow operations require admin authentication
  const ADMIN_TOKEN = process.env.MOOSE_ADMIN_TOKEN; // Set your admin bearer token
  
  // Trigger a workflow with admin authentication
  async function triggerDataPipeline(workflowName: string, inputData: any) {
    const response = await fetch(`http://localhost:4000/admin/workflows/${workflowName}/trigger`, {
      method: 'POST',
      headers: {
        'Authorization': `Bearer ${ADMIN_TOKEN}`,
        'Content-Type': 'application/json'
      },
      body: JSON.stringify(inputData)
    });
  
    if (!response.ok) {
      throw new Error(`Failed to trigger workflow: ${response.statusText}`);
    }
  
    return response.json();
  }
  
  // Query workflow execution history with admin authentication
  async function getWorkflowHistory(workflowName?: string) {
    const url = workflowName 
      ? `http://localhost:4000/admin/workflows/history?workflow_name=${workflowName}`
      : 'http://localhost:4000/admin/workflows/history';
  
    const response = await fetch(url, {
      headers: {
        'Authorization': `Bearer ${ADMIN_TOKEN}`
      }
    });
  
    if (!response.ok) {
      throw new Error(`Failed to fetch workflow history: ${response.statusText}`);
    }
  
    return response.json();
  }
  
  // Example usage
  await triggerDataPipeline('user-analytics-pipeline', { userId: '12345', event: 'signup' });
  const history = await getWorkflowHistory('user-analytics-pipeline');
  ```

  This example shows how to update existing workflow API calls to use the new admin-protected endpoints. All workflow trigger and history operations now require admin authentication via bearer token and use `/admin/` prefixed routes.

  PR: [#3432](https://github.com/514-labs/moose/pull/3432) | Docs: [/moosestack/workflows](https://docs.moosejs.com/moosestack/workflows)
