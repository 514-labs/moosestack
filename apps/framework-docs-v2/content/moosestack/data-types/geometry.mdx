# Geometry Types

ClickHouse provides geometry types for spatial data. Moose exposes type-safe helpers that map to the correct ClickHouse representations.

## Point

A single coordinate pair `[x, y]`.

<LanguageTabs>
  <LanguageTabContent value="typescript">
```typescript
import { ClickHousePoint } from "@514labs/moose-lib";

interface Location {
  coordinates: ClickHousePoint;    // Point → [number, number]
}
```
  </LanguageTabContent>
  <LanguageTabContent value="python">
```python
from moose_lib import Point

class Location(BaseModel):
    coordinates: Point    # tuple[float, float]
```
  </LanguageTabContent>
</LanguageTabs>

## Ring

A closed sequence of points forming a boundary (first and last points are the same).

<LanguageTabs>
  <LanguageTabContent value="typescript">
```typescript
import { ClickHouseRing } from "@514labs/moose-lib";

interface Boundary {
  outline: ClickHouseRing;    // Ring → Array<[number, number]>
}
```
  </LanguageTabContent>
  <LanguageTabContent value="python">
```python
from moose_lib import Ring

class Boundary(BaseModel):
    outline: Ring    # list[tuple[float, float]]
```
  </LanguageTabContent>
</LanguageTabs>

## Type Mapping Reference

| ClickHouse Type | TypeScript | Python |
|-----------------|------------|--------|
| `Point` | `ClickHousePoint` | `Point` (tuple[float, float]) |
| `Ring` | `ClickHouseRing` | `Ring` (list[tuple[float, float]]) |
| `LineString` | `ClickHouseLineString` | `LineString` (list[tuple[float, float]]) |
| `MultiLineString` | `ClickHouseMultiLineString` | `MultiLineString` (list[list[...]]) |
| `Polygon` | `ClickHousePolygon` | `Polygon` (list[list[...]]) |
| `MultiPolygon` | `ClickHouseMultiPolygon` | `MultiPolygon` (list[list[list[...]]]) |

<Callout type="info" title="Coordinate order">
Coordinates are represented as `[x, y]` pairs. For geographic data, this typically means `[longitude, latitude]`.
</Callout>
