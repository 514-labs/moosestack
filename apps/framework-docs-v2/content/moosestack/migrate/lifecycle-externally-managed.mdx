# Externally Managed

Use `EXTERNALLY_MANAGED` when another system owns schema changes.

## When to use

- CDC services create and evolve tables.
- A platform team owns warehouse schema.
- You only need typed models and query safety in Moose.

## Keep external models synced

```bash
moose db pull --olap-engine clickhouse --olap-url "$MOOSE_OLAP__CONNECTION"
```

## TypeScript example

```ts
import { OlapTable, LifeCycle, Key } from "@514labs/moose-lib";

interface ExternalUser {
  userId: Key<string>;
  email: string;
}

export const ExternalUsers = new OlapTable<ExternalUser>("external_users", {
  lifeCycle: LifeCycle.EXTERNALLY_MANAGED,
});
```

## Python example

```python
from pydantic import BaseModel
from moose_lib import Key, LifeCycle, OlapConfig, OlapTable

class ExternalUser(BaseModel):
    user_id: Key[str]
    email: str

external_users = OlapTable[ExternalUser](
    "external_users",
    OlapConfig(life_cycle=LifeCycle.EXTERNALLY_MANAGED),
)
```
