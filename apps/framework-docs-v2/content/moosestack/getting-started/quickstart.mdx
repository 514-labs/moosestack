# 5-Minute Quickstart

Start with the OLAP engine you plan to use.

## 1. Install CLI

```bash
bash -i <(curl -fsSL https://fiveonefour.com/install.sh) moose
```

## 2. Choose your engine

### Create project

```bash
moose init my-analytics-app typescript --olap-engine clickhouse
cd my-analytics-app
pnpm install
```

### Configure project

```toml title="moose.config.toml"
[olap]
engine = "clickhouse"
profile = "local"
connection = "http://panda:pandapass@localhost:18123/local"

[dev_infra]
profile = "local"
```

### Start development

```bash
moose dev --infra-profile local
```

### Plan and apply migrations

```bash
export MOOSE_OLAP__CONNECTION="http://panda:pandapass@localhost:18123/local"

moose generate migration --olap-engine clickhouse --olap-url "$MOOSE_OLAP__CONNECTION" --save
moose migrate --olap-engine clickhouse --olap-url "$MOOSE_OLAP__CONNECTION"
```

<Callout type="info" title="Snowflake is remote-first">
  Use remote credentials and run with a remote profile.
</Callout>

### Create project

```bash
moose init my-analytics-app typescript --olap-engine snowflake
cd my-analytics-app
pnpm install
```

### Configure project

```toml title="moose.config.toml"
[olap]
engine = "snowflake"
profile = "remote"
connection = "snowflake://<user>:<password>@<account>/<database>/<schema>?warehouse=<warehouse>&role=<role>"

[dev_infra]
profile = "remote_full"
```

### Start development

```bash
moose dev --infra-profile remote_full
```

### Plan and apply migrations

```bash
export MOOSE_OLAP__CONNECTION="snowflake://<user>:<password>@<account>/<database>/<schema>?warehouse=<warehouse>&role=<role>"

moose generate migration --olap-engine snowflake --olap-url "$MOOSE_OLAP__CONNECTION" --save
moose migrate --olap-engine snowflake --olap-url "$MOOSE_OLAP__CONNECTION"
```
