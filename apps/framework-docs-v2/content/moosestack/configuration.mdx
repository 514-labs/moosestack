# Project Configuration

MooseStack reads configuration from `moose.config.toml` and `MOOSE_*` environment variables.

## Precedence

Values resolve in this order:

1. System environment variables
2. `.env.local`
3. `.env.dev` or `.env.prod`
4. `.env`
5. `moose.config.toml`
6. Built-in defaults

## Canonical contract

```toml
[olap]
engine = "clickhouse" # clickhouse | duckdb | snowflake
profile = "remote"    # local | remote
connection = "https://user:password@host:8443/db"

[dev_infra]
profile = "remote_hybrid" # local | remote_hybrid | remote_full
```

## Environment variables

```bash
MOOSE_OLAP__ENGINE=clickhouse
MOOSE_OLAP__PROFILE=remote
MOOSE_OLAP__CONNECTION=https://user:password@host:8443/db
MOOSE_DEV_INFRA__PROFILE=remote_hybrid
```

## Security guidance

- Store credentials in secret managers or `.env.local`.
- Do not commit secrets to git.
- Use separate connection values for dev, staging, and production.
