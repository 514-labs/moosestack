# SQL Audit

## Coherence Issues

The two queries below answer similar questions ("how many planes are active?") but will return different numbers because they define "active" differently:

1. **Time window vs. snapshot**: The API/MCP query uses a rolling 2-minute window (`now() - INTERVAL 2 MINUTE`), which accumulates all aircraft seen across multiple polling cycles. The LLM query pins to a single `MAX(timestamp)`, which captures only the aircraft from the most recent fetch â€” a single 30-second polling snapshot. The rolling window will always return a higher count.

2. **Different tables**: The API/MCP query reads from `AircraftTrackingProcessedTable` while the LLM query reads from `AircraftTrackingDataTable`. These should contain the same aircraft (the processed table is derived 1:1 from the raw table), but querying different tables for the same metric is fragile â€” if the transform pipeline lags or filters records, the counts will diverge.

3. **DISTINCT vs. row count**: The API/MCP query uses `count(DISTINCT hex)` to deduplicate aircraft seen multiple times in the window. The LLM query uses `COUNT(DISTINCT hex)` for `total_aircraft` but then uses `SUM(CASE ...)` which counts *rows*, not distinct aircraft. If an aircraft appears in multiple rows at the same timestamp (e.g. duplicate messages), `planes_in_air + planes_on_ground` could exceed `total_aircraft`.

To make these consistent, they should use the same table, the same time filter, and the same deduplication strategy.

## Active Aircraft Count

Used by both the REST API (`GET /aircraft/active-count`) and the MCP tool (`planes_count`).

```sql
SELECT count(DISTINCT hex) AS active_aircraft
FROM AircraftTrackingProcessedTable
WHERE timestamp > now() - INTERVAL 2 MINUTE
```

| Item | Detail |
|------|--------|
| Source file (API) | `packages/moosestack-service/app/apis/aircraft.ts` |
| Source file (MCP) | `packages/moosestack-service/app/apis/mcp.ts` |
| Table | `AircraftTrackingProcessedTable` |
| Access mode | `readonly: "2"` |
| Filters | `timestamp > now() - INTERVAL 2 MINUTE` |
| Returns | Single row: `{ active_aircraft: number }` |

## Aircraft Status Breakdown (LLM-generated)

Generated by the frontend chat LLM via the `query_clickhouse` MCP tool.

```sql
SELECT
  COUNT(DISTINCT hex) as total_aircraft,
  SUM(CASE WHEN alt_baro_is_ground = false THEN 1 ELSE 0 END) as planes_in_air,
  SUM(CASE WHEN alt_baro_is_ground = true THEN 1 ELSE 0 END) as planes_on_ground
FROM AircraftTrackingDataTable
WHERE timestamp = (SELECT MAX(timestamp) FROM AircraftTrackingDataTable)
```

| Item | Detail |
|------|--------|
| Source | LLM-generated via chat UI |
| Table | `AircraftTrackingDataTable` |
| Filters | Latest timestamp only (correlated subquery) |
| Returns | Single row: `{ total_aircraft, planes_in_air, planes_on_ground }` |
